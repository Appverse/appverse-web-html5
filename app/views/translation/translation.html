Explanation

We provide i18n (Internationalization) and l10n (Localization) to be able to switch languages on the fly at runtime.

Date, Currency and Number are translated using tmhDynamicLocale plugin which uses angular-locale scripts.

Text is translated via angular-translate plugin: Official site

Just add the code snippets provided below to your Appverse project. Go to the path 'app/resources/i18n' and create a JSON file for the language that you want to support.

If the JSON file is not found for a specific locale, the text will be kept as it is, non translated. It allows to support localization but not internationalization. Japanese and Arabic are examples of what can be done.

Example: In the HTML there is a key named WELCOME. This key corresponds to the key found in the JSON file at the path 'app/resources/i18n'. You just have to assign the key with desired the value. For example, in a JSON file for Spanish translations you can write "WELCOME": "Bienvenido!" and in another JSON file for English, "WELCOME": "Welcome!".

Note: Please use the translate and translate-values attribute like in the examples for full compatibility.
