Performance Strategies in data loading

This demo shows different approaches to show a big chunk of data (more than 24000) from a REST endpoint.

No Caching

Time for render: {{ execTime }} ms.

        EXPLANATION
        In this case the function (RESTFactory#readList) used to retrieve a big chunk of data is not appropriate. This function
        has been designed to retrieve a tree of records with metadata about ancestors and related data in a single request.
        So, the big amount of data is increased its weight in 150% (average).
        Moreover, we do not use cache for the GET method.
        Evidently, it is not very efficient!

Using Cache

Time for render: {{ execTime }} ms.

        EXPLANATION
        In this case we use a different strategy, using caching and the AngularJS's $http service to avoid metadata retrieving.
        The initial data load is lightly better but the performance is dramatically improved in navigation across pages of data.

Concurrent Calling

Time for render: {{ execTime }} ms.

        EXPLANATION
        In this case we use a parallelized call to several REST resources, separately retrieving them an d combining in a
        single resource. This approach significatively improves the initial load of data.
        Moreover, navigation across data pages has a good performance. Maybe this is the best strategy to load
        a big chunk of data.
